7. หลักการและทฤษฎีที่เกี่ยวข้อง

การเรียนรู้เชิงลึก (Deep Learning) และ Computer Vision ศึกษาการทำงานของโครงข่ายประสาทเทียมแบบ Convolutional Neural Networks (CNN) โดยเลือกใช้สถาปัตยกรรม ResNet-18 ซึ่งมีความเหมาะสมสำหรับงานจำแนกคุณภาพจากภาพ เนื่องจากมีความแม่นยำสูงและใช้ทรัพยากรการประมวลผลน้อยกว่าสถาปัตยกรรมขนาดใหญ่ นำมาประยุกต์ใช้ในฟีเจอร์ตรวจสอบคุณภาพ AI ของระบบ FreshPro เพื่อวิเคราะห์สีและความสมบูรณ์ของรูปทรงเนื้อแต่ละชิ้น และให้คะแนนจัดเกรดตามเกณฑ์ที่โรงงานกำหนด (เกรด A–D) โดยบันทึกผลลัพธ์ทั้งหมดไว้เพื่อวิเคราะห์แนวโน้มคุณภาพการผลิตในระยะยาว

สถาปัตยกรรมส่วนติดต่อผู้ใช้งานแบบ Component-Based (Component-Based UI Architecture) ศึกษาแนวคิดการออกแบบส่วนติดต่อผู้ใช้โดยแบ่งออกเป็นหน่วยย่อยที่นำกลับมาใช้ซ้ำได้ (Reusable Components) โดยใช้ไลบรารี React 19 เป็นหลัก ทำให้แต่ละส่วนของระบบ เช่น Sidebar, Topbar, แบบฟอร์มรับวัตถุดิบ และตารางผลการตรวจสอบ ต่างแยกออกจากกันอย่างชัดเจน ง่ายต่อการพัฒนาและดูแลรักษา รวมถึงการใช้ React Hooks ได้แก่ useState, useEffect และ useMemo เพื่อจัดการสถานะและผลข้างเคียงภายใน Client Components

เฟรมเวิร์กเว็บแบบ Server-Side Rendering และ App Router (Next.js) ศึกษาการพัฒนาเว็บแอปพลิเคชันด้วย Next.js ซึ่งรองรับทั้ง React Server Components (RSC) สำหรับหน้าที่ไม่ต้องการ Interactivity เพื่อลดปริมาณ JavaScript ที่ส่งไปยังผู้ใช้งาน และ Client Components สำหรับส่วนที่ต้องการโต้ตอบแบบ Real-time เช่น หน้าควบคุมการผลิตและหน้าตรวจสอบคุณภาพ ทำให้ระบบมีประสิทธิภาพในการโหลดหน้าและรองรับการขยายตัวได้ดี

การจัดการข้อมูลข้ามหน้าผ่าน Web Storage API ศึกษาการใช้งาน localStorage ซึ่งเป็นกลไกจัดเก็บข้อมูลฝั่งไคลเอนต์ตามมาตรฐาน Web Storage API เพื่อเชื่อมโยงข้อมูลระหว่างหน้าต่าง ๆ ของระบบ ได้แก่ การส่งต่อข้อมูลใบสั่งซื้อ (sourcing_orders) ไปยังหน้ารับเข้าวัตถุดิบ (intake_records) และต่อเนื่องไปยังหน้าควบคุมการผลิต (production_lines) และหน้าตรวจสอบคุณภาพ (qc_records) ทำให้ข้อมูลไหลต่อเนื่องกันครบทุกขั้นตอนของกระบวนการผลิตโดยไม่จำเป็นต้องพึ่งพา Global State Management Library

บริการฐานข้อมูลและ Backend แบบ Backend-as-a-Service (BaaS) ศึกษาการใช้งาน Supabase ซึ่งเป็นแพลตฟอร์ม BaaS แบบ Open Source ที่ให้บริการฐานข้อมูล PostgreSQL พร้อม REST API และระบบยืนยันตัวตนในตัว เพื่อจัดเก็บข้อมูลหลักของระบบ อาทิ ข้อมูลผู้ใช้งาน ข้อมูลวัตถุดิบ และข้อมูลการผลิต โดยเชื่อมโยงกับ Python Flask API ซึ่งทำหน้าที่ประมวลผลและวิเคราะห์ข้อมูลเชิงสถิติก่อนส่งผลลัพธ์กลับไปแสดงบนเว็บแอปพลิเคชัน

การออกแบบส่วนติดต่อผู้ใช้งานแบบ Responsive Design ด้วย Utility-First CSS ศึกษาแนวคิด Responsive Web Design ตามหลักการของ W3C เพื่อให้ระบบสามารถปรับขนาดและรูปแบบการแสดงผลได้อย่างเหมาะสมกับอุปกรณ์ที่หลากหลาย โดยนำ Tailwind CSS v4 มาใช้ในการจัดการสไตล์ผ่านระบบ Design Token ที่กำหนดสีและขนาดของ UI ไว้เป็นมาตรฐานเดียวกันทั้งระบบ ทำให้การพัฒนา UI มีความสม่ำเสมอ ลดความซ้ำซ้อนของโค้ด และง่ายต่อการปรับปรุงในอนาคต
